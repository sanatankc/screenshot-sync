# Repo Working Rules

## Commit Timing

- When working through tasks in this repo, do not commit immediately after finishing a task. Leave the task changes uncommitted for review.
- Before starting a new task, always do a git status to ensure if there is anything to commit from previous task. if it's clear to you that uncommited changes are from a previous task you were doing, commit them. other wise ask the user for clarity.
- Before starting a new task, it is not only a good to commit to the previous task, but also to mark the previous task as complete in the dex tasks list.
